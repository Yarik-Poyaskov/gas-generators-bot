# ⚡ GPU Checklist Bot & Energy Portal

Комплексна система корпоративного рівня для автоматизації збору, верифікації та моніторингу даних газопоршневих установок (ГПУ) мережі ТЦ "Епіцентр К". Проєкт поєднує в собі Telegram-бота для збору оперативної звітності на базі Finite State Machine (FSM) та сучасний веб-інтерфейс (Next.js 16 / Tailwind CSS 4) для візуалізації статусу об'єктів у реальному часі.

---

## 🚀 Основний функціонал

### 🤖 Telegram Бот (`@gpu_checklist_bot`)
*   **Повний чек-лист**: Інтерактивний покроковий процес введення технічних параметрів роботи установок з валідацією типів (підтримка `float` та нульових значень `0` / `0.0`).
*   **Гнучкі режими роботи**: Вибір режиму ("Працює", "Не працює", "Аварія"). При виборі режиму "Не працює" з'являється кнопка **"⏩ Пропустити"** для параметрів, що не стосуються зупиненої установки.
*   **Розумна обробка тиску**: Окремий та розділений запуск з повідомленнями для 4-х контурів тиску антифризу (малий та великий контури, до та після запуску ГПУ).
*   **Фото-фіксація**: Обов'язкове завантаження фотографій мультиметра та ЩОС для підтвердження внесених даних.
*   **Керування змінами (Shifts)**: Логіка заступання та здачі зміни операторами з відображенням реального ПІБ, автоматичними нагадуваннями про завершення зміни та автозакриттям після налаштованого інтервалу.
*   **Система Опитувань (Survey 1)**: Збір фото та коментарів безпосередньо в групах об'єктів з блокуванням опитування першим відповілим та вивантаженням результатів у Google Sheets.
*   **Таргетована розсилка (Broadcast)**: Створення розсилок в адмін-панелі з вибором конкретних груп об'єктів, можливістю прикріплювати будь-які документи (PDF, Word, Excel) нарівні з зображеннями та предпросмотром перед надсиланням.
*   **Адміністрування безпосередньо з бота**: Інлайн-меню для редагування параметрів звітів, коригування значень, налаштування розкладу надсилання звітів та інтервалів перевірок.

### 🌐 Energy Portal (Web Dashboard)
*   **Live Status**: Моніторинг усіх ГПУ на одній сторінці з миттєвим оновленням через WebSockets.
*   **Автонаслідування статусів ГПУ**: Автоматична передача останнього статусу, режиму роботи, автора та потужності зі вчорашнього дня на сьогодні для безперервного графіка роботи на стыку суток (24:00 -> 00:00), якщо звіт ще не заповнено оператором.
*   **Редизайн індикаторів**: Стильні градієнтні кольори смуги потужності (синій градієнт за звичайних умов, оранжево-червоний при критичному навантаженні >90%) зі збільшеною товщиною для покращеної читаності.
*   **PWA (Progressive Web App)**: Можливість встановлення сайту як додатка на смартфон із власними синьо-жовтими градієнтними іконками та фоновим кэшуванням ресурсів за стратегією *Stale-While-Revalidate* через Service Worker.
*   **Web Push Сповіщення**: Сповіщення в реальному часі про аварії ГПУ на підключених пристроях.
*   **Портал Трейдера**: Веб-інтерфейс для публікації графіків роботи, імпорту розкладів та парсингу повідомлень з Telegram з покращеною фільтрацією стопів (виключення помилкових спрацювань на дефісах у назвах об'єктів типу `KC GPU-1`).
*   **Історія та аналітика**: Таблиця звітів з пагінацією, фільтрацією та можливістю експорту даних в Excel (`.xlsx`) безпосередньо з фронтенду через API.
*   **Безпека**: OTP авторизація через Telegram, JWT токени доступу, захист від дублювання розкладів (Trader Schedules).

## 📋 Покроковий чек-лист Telegram-бота (12 кроків)

При виборі опції **"Заповнити Чек-Лист"** бот послідовно проводить оператора через наступні кроки:

1.  **Режим роботи ГПУ**: Вибір із кнопок (`Працює`, `Не працює`, `Аварія`).
2.  **Час запуску ГПУ**: Інлайн-клавіатура для вибору години та хвилин (доступна кнопка `⏩ Пропустити` у режимі "Не працює").
3.  **Навантаження ГПУ**: Розділений ввід:
    *   *3.1 Потужність (%)* - числове значення.
    *   *3.2 Потужність (кВт)* - числове значення.
4.  **Напруга АКБ стартера**:
    *   *4.1 Напруга АКБ 1 (В)* - числове значення.
    *   *4.2 Напруга АКБ 2 (В)* - числове значення.
5.  **Напруга батареї Haas**: Напруга внутрішньої батареї контролера Haas (В).
6.  **Тиск оливи**: Тиск оливи в системі двигуна (бар).
7.  **Температура охолоджуючої рідини**: Температура антифризу на виході з двигуна (°C).
8.  **Напрацювання (мотогодини)**: Поточні моточасові показники роботи ГПУ (г).
9.  **Ліміт змащення підшипника**: Моточасовий показник до наступного змащення підшипника генератора (г).
10. **Тиск малого контуру (інтеркулер)**:
    *   *10.1 До пуску ГПУ (GK/MCC, до насоса)* - тиск в контурі інтеркулера до пуску (бар, є кнопка `⏩ Пропустити`).
    *   *10.2 Після пуску ГПУ (GK/MCC, до насоса)* - тиск в контурі інтеркулера після пуску (бар, є кнопка `⏩ Пропустити`).
11. **Тиск великого контуру (двигун)**:
    *   *11.1 До пуску ГПУ (NK/ECC, до насоса)* - тиск в контурі двигуна до пуску (бар, є кнопка `⏩ Пропустити`).
    *   *11.2 Після пуску ГПУ (NK/ECC, до насоса)* - тиск в контурі двигуна після пуску (бар, є кнопка `⏩ Пропустити`).
12. **Фото-фіксація**: Послідовне надсилання фотографій мультиметра та ЩОС.

├── scratch/                    # Допоміжні скрипти розробника (відладка)

## 🔧 Розгортання та Налаштування

### 1. Системні вимоги
*   Python 3.10+
*   Node.js 20+
*   Права локального адміністратора (опціонально, для точного логування та портів)

### 2. Налаштування середовища (`.env`)
Створіть файл `.env` у кореневій директорії проєкту за наступним зразком:

```env
# --- Налаштування Telegram-бота ---
BOT_TOKEN=your_telegram_bot_token
ADMIN_IDS=[123456789,987654321]              # JSON-список або через кому 123,456
GROUP_ID=-1001234567890                        # Основний чат для відправки звітів
SPECIAL_GROUP_ID=-1001234567890                 # Чат для спрощених звітів
OBH_SPECIAL_GROUP_ID=-1001234567890            # Виділений канал для звітів OBH GPU
TEST_SPECIAL_GROUP_ID=-1001234567890           # Група тестування картинок звітів
SUMMARY_REPORT_TIME=09:00                      # Час надсилання щоденного зведеного звіту

# --- База даних ---
DB_URL=sqlite+aiosqlite:///reports.db
TC_NAME="ТЦ Назва"
USERS_PER_PAGE=10

# --- Парсинг Трейдера ---
MONITORED_TRADER_IDS=[123456789,987654321]    # ID трейдерів для відстеження
TRADER_MONITOR_GROUP_ID=-1001234567890            # Чат моніторингу графіків трейдерів

# --- Web Push (VAPID ключі) ---
VAPID_PUBLIC_KEY=your_vapid_public_key
VAPID_PRIVATE_KEY=your_vapid_private_key
VAPID_CLAIM_EMAIL=your_email@example.com

# --- Налаштування Web API ---
API_TOKEN="your_secure_api_token"
JWT_SECRET="your_jwt_secret_key"
CORS_ORIGINS=http://localhost:3000

# --- Інтеграція Google Sheets & Drive ---
GOOGLE_SHEETS_JSON_KEY=service_account.json
GOOGLE_SHEETS_URL=https://docs.google.com/spreadsheets/d/your_spreadsheet_id/edit
GOOGLE_APPS_SCRIPT_URL=https://script.google.com/macros/s/your_script_id/exec
GOOGLE_APPS_SCRIPT_TOKEN=your_secret_upload_token
```

> [!WARNING]
> Переконайтеся, що `GOOGLE_SHEETS_JSON_KEY` вказує на коректний файл сервісного акаунта у корені проєкту, якщо увімкнено вивантаження звітів.

### 3. База даних та міграція

При першому запуску файл бази даних `reports.db` створюється автоматично. Проте, якщо схема бази даних зазнала змін, необхідно застосувати міграції:

```powershell
# Додавання нових полів чек-ліста (тиск та ліміти)
python migrate_add_new_fields.py

# Створення таблиці смен (shifts)
python migrate_shifts.py

# Налаштування інтервалів нагадувань смен
python migrate_shift_interval.py
```

> [!IMPORTANT]
> Якщо вам потрібно повністю очистити схему бази даних для запуску з нуля, просто видаліть файл `reports.db`:
> `Remove-Item -Path "reports.db" -ErrorAction SilentlyContinue` (PowerShell) або `del reports.db` (CMD).

### 4. Запуск бекенду (Бот + API Server)
Усі залежності встановлюються через `requirements.txt`. Бот та FastAPI сервер працюють в одному циклі подій, що забезпечує стабільну відправку WebSocket-повідомлень.

```bash
# Встановлення залежностей Python
pip install -r requirements.txt

# Запуск системи
python bot.py
```

### 5. Запуск фронтенду (Next.js Dashboard)
Створіть у папці `/frontend` файл `.env.local` з URL-адресою бекенду:
```env
NEXT_PUBLIC_API_URL=http://localhost:7895
NEXT_PUBLIC_WS_URL=ws://localhost:7895
```

Виконайте збірку та запуск:
```bash
cd frontend

# Встановлення NPM залежностей
npm install

# Запуск у режимі розробки
npm run dev

# Збірка продакшн-версії
npm run build
npm run start
```

## 📈 Google Apps Script (GAS) для вивантаження фотографій

Щоб обійти ліміт вільного дискового простору сервісних акаунтів Google (0 байт), завантаження фотографій реалізовано через персональний аккаунт за допомогою Google Apps Script. 

1. Створіть новий проект Google Apps Script у вашому Google Drive.
2. Вставте код скрипта, який приймає `POST` запити з токеном `GOOGLE_APPS_SCRIPT_TOKEN`.
3. Опублікуйте скрипт як веб-додаток (доступ: "Anyone").
4. Скопіюйте отриманий URL веб-додатка та вставте його у змінну `GOOGLE_APPS_SCRIPT_URL` у вашому файлі `.env`.

Бот автоматично створить папку `Survey_Photos` на Google Диску і буде структурувати фото по папках тем опитувань із формулами прямого перегляду у таблиці: `=HYPERLINK("view_url"; IMAGE("download_url"))`.
