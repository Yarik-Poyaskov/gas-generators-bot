# ⚡ GPU Checklist Bot & Energy Portal

Комплексна система для автоматизації збору та моніторингу даних газопоршневих установок (ГПУ) мережі ТЦ "Епіцентр К". Проєкт поєднує в собі Telegram-бота для збору оперативної звітності та сучасний веб-інтерфейс для візуалізації статусу об'єктів у реальному часі.

## 🚀 Основний функціонал

### 🤖 Telegram Бот (@gpu_checklist_bot)
- **Повний чек-лист**: Покрокове введення технічних параметрів (навантаження, напруга АКБ, мотогодини, тиск оливи тощо).
- **Статус ГПУ (Короткий звіт)**: Швидке оновлення поточного стану та потужності.
- **Фото-фіксація**: Обов'язкове завантаження фото мультиметра та ЩОС для підтвердження даних.
- **Керування змінами**: Система заступання та здачі зміни операторами з автоматичними нагадуваннями та ПІБ-ідентифікацією.
- **Інтеграція з групами**: Автоматична розсилка графіків трейдерів у локальні групи об'єктів та збір підтверджень.

### 🌐 Energy Portal (Web Dashboard)
- **Live Status**: Моніторинг усіх ГПУ на одній сторінці з оновленням через WebSockets.
- **Інтелектуальна сітка**: Адаптивний інтерфейс (4/5/6 колонок) зі Smart Sidebar.
- **Web Push Сповіщення**: Миттєві браузерні повідомлення при виникненні аварій на об'єктах.
- **Портал Трейдера**: Веб-інтерфейс для парсингу та публікації графіків роботи.
- **Безпека**: Захист через JWT токени, OTP авторизація через Telegram та проксування через Cloudflare Tunnel.

## 🛠 Технологічний стек

- **Backend**: Python 3.10+, Aiogram 3.x (Bot), FastAPI (API), APScheduler (Планувальник).
- **Database**: SQLite + Aiosqlite (асинхронна робота з БД).
- **Frontend**: Next.js 14, React, Tailwind CSS, Framer Motion, Lucide Icons.
- **Infrastructure**: Cloudflare Tunnel (безпечний доступ), Service Workers (PWA & Push).
- **Integrations**: Google Sheets API (дублювання звітів).

## 📦 Встановлення та запуск

### 1. Клонування репозиторію
```bash
git clone https://github.com/your-username/gpu-checklist-bot.git
cd gpu-checklist-bot
```

### 2. Налаштування середовища
Створіть файл `.env` у корені проєкту (використовуйте приклад з документації):
```env
BOT_TOKEN=your_token
ADMIN_IDS=[123, 456]
GROUP_ID=-100...
DB_URL=sqlite+aiosqlite:///reports.db
```

### 3. Запуск Backend (Бот + API)
```bash
pip install -r requirements.txt
python bot.py
```

### 4. Запуск Frontend (Dashboard)
```bash
cd frontend
npm install
npm run dev
```

## 📁 Структура проєкту
- `/app` — Логіка бота, БД та API сервера.
- `/frontend` — Веб-додаток на Next.js.
- `bot.py` — Точка входу для запуску всієї системи.
- `mapping.json` — Налаштування розпізнавання об'єктів для трейдера.
